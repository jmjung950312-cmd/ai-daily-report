## ⭐ 핵심 변화 TOP 3

### 1. Claude Code v2.1.126 — 현실 버그 대규모 패치

**📌 무슨 일?**

5월 1일, Claude Code v2.1.126이 배포됐어. 이번 릴리즈는 신기능보다 **"있으면 안 되는 버그들"**을 한꺼번에 잡은 게 핵심이야.

패치된 주요 버그:

1. **Mac 수면 후 Stream idle timeout** — 맥북을 닫았다가 열었을 때 Claude Code 연결이 끊기던 문제 수정. 긴 모델 사고(extended thinking) 중에 타임아웃이 발생하던 버그도 동시에 잡혔어.

2. **Windows CJK 텍스트 깨짐** — 일본어·한국어·중국어로 입력하면 글자가 깨져서 나오던 버그 수정. Windows에서 한국어로 Claude Code를 쓰는 사람이라면 이번 업데이트가 가장 반가울 거야.

3. **Windows 클립보드 보안 취약점** — 복사한 내용이 프로세스 명령줄 인수에 노출되어 EDR/SIEM(기업 보안 모니터링 도구 — 회사 PC에서 뭘 실행하는지 기록하는 소프트웨어) 에 보이던 문제 수정. 22KB 이상 선택이 클립보드에 안 들어가는 버그도 함께 수정.

4. **2000px 이상 이미지 붙여넣기** — 큰 스크린샷을 붙여넣으면 세션 자체가 깨지던 버그. 이제 자동으로 축소 후 붙여넣기.

5. **iTerm2 + tmux 터미널 티어링** — 터미널 알림 전송 시 화면에 랜덤 문자가 떠오르거나 입력 위치가 밀리던 현상 수정.

신기능도 함께 들어왔어:

- **Model picker 게이트웨이 지원** — `ANTHROPIC_BASE_URL`을 회사 게이트웨이 주소로 설정하면 /model 선택 화면에서 그 게이트웨이의 모델 목록이 자동으로 표시돼.
- **OAuth WSL2·SSH·컨테이너 개선** — Windows Subsystem for Linux, SSH 터널, 도커 컨테이너 환경에서 Claude Code 로그인이 더 안정적으로 동작해.
- **Windows PowerShell 7 자동 감지** — Microsoft Store, MSI, .NET 글로벌 툴로 설치된 PowerShell 7을 자동으로 인식.

**🤔 왜 중요?**

두 가지야:

1. **Mac 수면 버그는 매일 맞닥뜨리는 불편이었어** — 맥북 사용자라면 회의 갔다 돌아와서 Claude Code가 끊긴 채로 있는 경험을 해봤을 거야. 이게 수정됐다는 것만으로도 체감이 크게 달라져.

2. **Windows 한국어 사용자에게 직접 영향** — CJK 깨짐은 Windows에서 Claude Code를 쓰는 한국어 사용자에게 사용 자체를 막는 수준의 버그였어. 이번에 잡혔으니 Windows에서도 편하게 쓸 수 있게 됐어.

**👉 너에게 영향**

- **업데이트 방법**: `npm install -g @anthropic-ai/claude-code@latest` 또는 Claude Code 앱 자동 업데이트
- **Mac 사용자**: 이제 맥북 화면을 닫았다 열어도 세션이 유지될 거야. 타임아웃 때문에 /resume를 자주 쓰던 패턴이 줄어들 것.
- **이미지 작업**: 스크린샷을 자주 붙여넣는 스타일이라면 고해상도 이미지가 이제 세션을 깨뜨리지 않아.

부대 조건:

- **Amazon의 OpenAI 투자** $50B [unverified: 정확 금액 미확인] — 다년간 Trainium 칩 2기가와트 인프라 계약 포함
- **API 가격**: GPT-5.5 기준 입력 $5 / 출력 $30 per MTok

**🤔 왜 중요?**

비유하면 — 7년간 "코카콜라는 맥도날드에서만"이었는데 갑자기 버거킹에도 들어온 것과 같아.

2019년 Microsoft의 OpenAI 투자로 시작된 독점적 파트너십은 Azure OpenAI Service라는 형태로 굳어져 있었어. 기업이 GPT 모델을 쓰려면 거의 무조건 Azure를 써야 했어.

이번 AWS 파트너십이 바꾸는 세 가지:

1. **기업 고객 선택지 확대** — Azure 인프라가 싫었던 기업들이 Bedrock을 통해 GPT 모델을 쓸 수 있게 됐어.
2. **클라우드 AI 레이어 평준화** — AWS, Azure, Google Cloud 모두 GPT 모델을 제공하는 시대. 클라우드 선택이 AI 모델 선택에서 분리됐어.
3. **경쟁 구도 변화** — Bedrock에서 Claude(Anthropic)와 GPT(OpenAI)가 나란히 제공돼. 기업이 같은 인프라에서 두 모델을 비교할 수 있어.

**👉 너에게 영향**

- **현재 Claude API 사용자**: 직접 영향 없음. 하지만 같은 AWS Bedrock 안에서 Claude·GPT를 비교 테스트하는 환경이 생긴 거야.
- **AWS 기반 프로젝트 있다면**: Bedrock에서 GPT-5.4를 직접 호출하는 게 이제 가능해. 단일 API 엔드포인트로 두 모델을 전환 테스트할 수 있어.
- **무료 대안**: AWS 무료 크레딧 범위에서 Bedrock에 접근 가능. $0.30 선에서 몇 번 테스트는 가능해.

### 3. Anthropic 1M 컨텍스트 베타 헤더 종료 — 구버전 모델 사용자 필독

**📌 무슨 일?**

Anthropic이 Claude Sonnet 4.5와 Claude Sonnet 4(구버전)에서 `context-1m-2025-08-07` 베타 헤더를 비활성화했어 [unverified: 정확 날짜 미확인, 5월 초 시행].

이게 무슨 말이냐면:

- 이전에는 API 요청 헤더에 이 값을 추가하면 최대 1M(100만) 토큰까지 컨텍스트를 넣을 수 있었어.
- 이제 그 헤더를 넣어도 아무 효과가 없어. 200K 토큰(20만)을 초과하는 요청은 에러가 돌아와.
- **1M 컨텍스트를 계속 쓰고 싶다면** → Claude Sonnet 4.6 또는 Claude Opus 4.6으로 업그레이드. 이 두 모델은 별도 헤더 없이 1M이 기본 지원돼.

**🤔 왜 중요?**

API를 통해 대용량 문서나 긴 코드베이스 분석에 Claude를 쓰는 사람에게는 조용한 Breaking Change야.

비유하면 — 공용 주차장이 "임시 무제한 이용"에서 "공식 지원 모델로 이동하세요"로 바뀐 거야. 1M 컨텍스트가 "실험적 기능"에서 "최신 모델의 기본 기능"으로 전환 완료된 거야.

Anthropic의 1M 컨텍스트 전략 흐름:

| 시점 | 내용 |
|---|---|
| 2025-08 이전 | 200K가 최대 컨텍스트 |
| 2025-08 | 베타 헤더로 1M 지원 시작 (Sonnet 4, Sonnet 4.5 대상) |
| 2026-04~05 | Sonnet 4.6, Opus 4.6에서 베타 헤더 없이 1M 정식 지원 |
| 현재 | 구버전 모델에서 베타 헤더 종료 → 신버전으로 마이그레이션 유도 |

**👉 너에게 영향**

영향받는 사람은 다음 두 조건을 모두 충족하는 경우야:

1. Anthropic API를 직접 사용 (Claude.ai Pro 플랜 내 사용자는 해당 없음)
2. API 요청에 `anthropic-beta: context-1m-2025-08-07` 헤더를 넣은 경우

해당된다면: 코드에서 이 헤더를 제거하고 모델을 `claude-sonnet-4-6` 또는 `claude-opus-4-6`으로 변경. 가격은 표준 요금 그대로야.

## 🔬 트렌드 한 컷: MCP가 특정 회사 기술에서 "공통 표준"으로 전환되다

Anthropic이 2024년 11월 공개한 **Model Context Protocol(MCP)** 이 2026년 3월 기준 **9700만 설치**를 돌파했어. 그리고 이제 **Linux Foundation**이 MCP의 오픈 거버넌스를 맡기로 했어 [unverified: 정확 날짜 미확인].

**MCP가 하는 일**: AI 에이전트가 데이터베이스를 조회하고, 외부 도구를 부르고, 다른 에이전트와 협력하는 방법을 정해놓은 규격이야. 비유하면 "에이전트 간 통역 표준어"야.

**왜 이 전환이 중요한가?**

HTML을 생각해봐. 처음엔 Tim Berners-Lee 개인이 만든 규격이었다가 W3C(월드와이드웹 컨소시엄)가 표준화하면서 "웹의 공통 언어"가 됐잖아. MCP의 Linux Foundation 이관은 그것과 같은 구조야.

"Anthropic이 만든 것" → "AI 에이전트의 공통 언어"로 넘어가는 공식 전환점이야.

**숫자로 보는 MCP 확산**:

| 지표 | 수치 |
|---|---|

**이 시점에 중요한 이유**:

AI 에이전트 시대의 패권은 "어떤 모델이 똑똑한가"뿐만 아니라 "어떤 표준 위에서 에이전트들이 연결되는가"에도 달려있어.

Anthropic이 MCP를 오픈 거버넌스로 이관한 건 단기 수익보다 장기 생태계를 선택한 거야. Claude가 MCP를 만든 회사라는 브랜드 혜택은 유지하면서, 모든 AI 도구가 따르는 표준이 됨으로써 얻는 생태계 영향력이 더 크다고 판단한 셈이야.

## 💡 활용 아이디어

### ⏱️ 15분으로 시도

1. **Claude Code v2.1.126 업데이트 + 수면 테스트**: `npm install -g @anthropic-ai/claude-code@latest` 실행 후 맥북 화면을 5분 닫았다가 열어봐. 세션이 유지된다면 버그가 잡힌 거야. 업데이트만으로 매일의 불편이 사라질 가능성이 높아.

2. **API 코드에서 context-1m 헤더 확인**: 현재 Anthropic API를 호출하는 코드에서 `context-1m-2025-08-07` 문자열을 검색해봐. 있다면 지금 당장 에러가 나지 않더라도, 모델을 `claude-sonnet-4-6`으로 업그레이드해두는 게 안전해.

### 🕐 1시간으로 시도

3. **MCP 서버 탐색 및 연결**: Claude Code가 MCP를 지원한다는 건 이미 알지? 이제 공개 MCP 서버만 10,000개가 넘어. `mcp.so` 또는 GitHub `awesome-mcp-servers`에서 현재 사이드 프로젝트에 바로 붙일 수 있는 MCP 서버를 찾아봐. 예: Supabase MCP, GitHub MCP, Google Drive MCP. 연결 자체는 설정 파일 5줄이면 돼.

4. **AWS Bedrock에서 GPT-5.4 첫 테스트**: AWS 계정이 있다면 Amazon Bedrock 콘솔에서 GPT-5.4를 직접 호출해봐. Claude Sonnet 4.6과 같은 프롬프트로 비교 테스트해보면, 어떤 유형의 작업에서 어느 모델이 더 나은지 직접 감이 생겨.

### 📅 하루로 시도

5. **터미널 AI 도구 3종 비교**: Claude Code(v2.1.126), Codex(0.128.0), Gemini CLI를 같은 작업에 모두 써보는 하루를 만들어봐. 특히 `goal 지속성(Codex)`과 Claude Code의 `/loop`를 비교하면, 각 도구의 "장기 작업 지속성"에서 뚜렷한 차이가 보여. 비교 메모 하나 남겨두면 나중에 어떤 도구를 어떤 상황에서 쓸지 판단 기준이 생겨.
