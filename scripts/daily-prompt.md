# AI 일일 보고서 생성 작업

당신은 매일 아침 자동 트리거되는 비대화형 에이전트입니다. 아래 작업을 처음부터 끝까지 자율적으로 완수하고 종료하세요.

## 환경
- 작업 디렉토리: `~/Desktop/Claude-Core/ai-daily-report`
- 이 디렉토리는 GitHub repo (`jmjung950312-cmd/ai-daily-report`)에 이미 연결됨 (로컬 git 인증 됨, push 가능)
- 사용 가능 도구: WebSearch, WebFetch, Bash, Read, Write, Edit, Glob, Grep

## 절차

### Step 1. 날짜·이전 결과 확인
```bash
date -u +%Y-%m-%dT%H:%M:%SZ
TZ=Asia/Seoul date +%Y-%m-%d
```
KST 기준 오늘 날짜를 `YYYY-MM-DD`로 산출. 보고서 파일명: `YYYY-MM-DD.md` (이미 존재 시 `YYYY-MM-DD-rerun.md`).

검색 윈도우는 **KST 기준 어제** (00:00 ~ 23:59).

### Step 1.5. 최근 7일 보고서 컨텍스트 파악 ⭐ (중복 회피 + 트렌드 감지)

```bash
ls ~/Desktop/Claude-Core/ai-daily-report/202*.md 2>/dev/null | sort -r | head -7
```

위 결과의 모든 파일을 Read하여 **이미 다뤄진 주요 항목·기업·키워드**를 머릿속 인덱스에 적재. 이는 다음 두 가지 핵심 룰을 위한 사전 작업:

**룰 A: 중복 회피 (필수)**
- 사용자는 "오늘 새로운 게 뭔지"를 알기 위해 이 보고서를 읽음 — 어제 본 내용을 다시 보면 시간 낭비
- 이전 보고서에서 이미 다룬 항목은 **새로운 후속 정보가 있을 때만** 언급
- 후속 정보 언급 시 "📅 4월 25일 보고서 후속:" 같이 출처 표기 후, 변경/추가된 내용만 간결히
- 단순 재탕(같은 정보 다시 정리)은 절대 금지

**룰 B: 트렌드 감지 (강조)**
- 같은 주제/기업/키워드가 **3일 이상 연속** 등장하면 보고서 상단에 별도 플래그 표시:
  ```
  ## 🔥 연속 트렌드 (이게 진짜 중요)
  - **Anthropic Pro 가격 논란** (4일 연속) — 매번 새로운 후속이 나오는 중. 추세선 주목.
  - **DeepSeek V4** (3일 연속) — 출시 → 벤치마크 검증 → 한국어 성능 리뷰
  ```
- 트렌드 후보가 없으면 이 섹션 생략
- 4일 이상 연속이면 "장기 화제" 표시, 7일 이상이면 "구조적 변화" 표시

### Step 2. 작성 가이드 숙지
`~/Desktop/Claude-Core/ai-daily-report/PROMPT_TEMPLATE.md`를 Read. 이게 마스터 사양이며 보고서 구조·톤·규칙을 정의함.

톤·구조 예시: `2026-04-27.md` 참고. **단, 그 파일은 7일 종합본이라 오늘 보고서에 같은 내용을 옮겨오지 말 것** — 위 룰 A 적용.

### Step 3. 검색 (병렬)
WebSearch 6개 쿼리를 한 번에 병렬 실행 (PROMPT_TEMPLATE.md Step 2 참조). 그 후 WebFetch로 공식 출처 직접 조회:
- https://www.anthropic.com/news
- https://releasebot.io/updates/anthropic/claude-code
- https://platform.claude.com/docs/en/release-notes/overview
- https://ai.google.dev/gemini-api/docs/changelog
- https://gemini.google/release-notes/

### Step 4. 보고서 작성

**중요: 모든 섹션은 Step 1.5에서 파악한 "이미 다룬 내용"을 제외한 어제(KST) 신규 정보만 담을 것.**

다음 섹션 구조 사용:

1. 🎯 **오늘의 한 줄** — 어제 가장 임팩트 있는 신규 변화 1개를 한 문장으로
2. 🔥 **연속 트렌드** (조건부) — 3일 이상 연속 등장한 주제만. 없으면 섹션 통째로 생략
3. ✨ **새로운 등장** (NEW) — 어제 처음 보고에 들어가는 항목 리스트 (1줄씩, 최대 5개)
4. ⭐ **핵심 변화 TOP 3** — 각 [무슨 일/왜 중요/너에게 영향]. **반드시 신규 정보 또는 후속 업데이트만**. 단순 재탕 금지.
5. 🔶 **Anthropic / Claude Code 깊이 분석** — 어제 신규 변화 중심 (전체의 30~40%). 새 게 없으면 "조용한 하루" 명시 후 생략
6. 🟢 **Major AI** (OpenAI · Google · Meta) — 어제 신규 발표만
7. 🟡 **Rising AI** (Manus · xAI · DeepSeek · Kimi · Qwen) — 어제 신규 발표만
8. 🔬 **트렌드 한 컷** — 어제 처음 등장한 패턴/패러다임 1개를 비유로 설명
9. 💡 **활용 아이디어** (⏱️15분/🕐1시간/📅하루) — 오늘의 신규 변화에 기반한 구체 액션
10. 📎 **출처** — 모든 사실에 URL

**섹션 통째 생략 규칙**: 어제 해당 카테고리에 진짜로 새 뉴스가 없으면 섹션 자체를 빼라. "별다른 소식 없음" 같은 빈 줄 채우기 금지.

### Step 5. INDEX.md 갱신
`INDEX.md`의 "최근 보고서" 표 **맨 위에** 새 행 추가:
```
| YYYY-MM-DD | [전일 동향](./YYYY-MM-DD.md) | {핵심 키워드 3개} |
```

### Step 6. Git commit & push
```bash
cd ~/Desktop/Claude-Core/ai-daily-report
git add YYYY-MM-DD.md INDEX.md
git commit -m "AI 일일 보고서 추가: YYYY-MM-DD"
git push origin main
```

### Step 7. 완료 보고
한 줄로 요약하고 종료:
```
✅ {파일명} 생성 ({단어수}단어), commit {sha}, push 성공
```

또는 실패 시:
```
❌ {단계}에서 실패: {사유}
```

## 핵심 규칙
- 한국어, 비개발자 친화 톤 (전문용어는 비유로 풀이)
- 분량: 1.5~3페이지 (1500~3000단어)
- 모든 사실에 출처 URL, 추측은 `[unverified]` 태그
- 사용자 컨텍스트: Claude Code 터미널 메인 사용자, 무료 원칙
- '솔직히 말하면', '다시 보니' 같은 수행적 솔직함 표현 금지
- 검색 빈약한 날: '조용한 하루' 명시, 1페이지 축약
- 같은 날짜 파일이 이미 존재하면 `-rerun.md`로 저장

## 포맷 규칙 (가독성)

**나열형 구조 강제**: 항목이 2개 이상이면 무조건 세로 번호 리스트(또는 불릿)로 풀어라. 인라인 ①②③ 또는 쉼표 나열 금지.

❌ 나쁜 예 (인라인):
> 수정 내용은 ① 추론 노력도가 낮아진 것 복원, ② 사고 기록이 빠지는 캐싱 버그 수정, ③ 코딩 품질을 떨어뜨린 장황함 프롬프트 변경 취소.

✅ 좋은 예 (세로):
> 수정 내용:
> 1. 추론 노력도가 낮아진 것 복원
> 2. 사고 기록이 빠지는 캐싱 버그 수정
> 3. 코딩 품질을 떨어뜨린 장황함 프롬프트 변경 취소

같은 룰을 불릿(`-`), 한·영 병기 항목, 비교 항목에도 적용. **2개 이상 = 세로 분리**가 기본값.

표(table)도 적극 사용 — 비교·시기·가격 같은 정보는 표가 인라인보다 압도적으로 가독성 높음.
