## ⭐ 핵심 변화 TOP 3

### 1. Anthropic-AWS: Claude Cowork → Amazon Bedrock 통합, AgentCore CLI 공개

**📌 무슨 일?**

4월 27일 AWS 위클리 라운드업에서 두 가지가 동시 발표됐어:

- **Claude Cowork on Amazon Bedrock 공개**: Anthropic이 만든 협업 AI 서비스가 AWS 클라우드에서 직접 배포 가능해졌어. 기업이 자체 AWS 환경 안에서 Claude를 "팀원처럼" 쓰는 구조가 공식화됐어.
- **Amazon Bedrock AgentCore CLI 공개**: 에이전트 앱 개발·배포를 CLI(명령줄 도구)로 관리하는 새 AWS 도구. Claude Code처럼 터미널에서 에이전트를 컨트롤하는 개발자 워크플로우를 AWS가 직접 지원하겠다는 선언.

**🤔 왜 중요?**

지금까지 AWS에서 Claude를 쓰려면 Amazon Bedrock을 통해 API를 직접 호출하는 방식이었어.

Claude Cowork Bedrock 통합은 이를 넘어서 — "협업 기능 통째로 AWS에서 실행"하는 문을 연 거야. AgentCore CLI는 "Claude Code가 로컬 개발자용이라면, AgentCore는 클라우드 엔터프라이즈용"이라는 포지셔닝.

**👉 너에게 영향**

- 사이드 프로젝트에 당장 영향은 없어. 하지만 장기적으로 **"내 프로젝트를 AWS에 올리면 Claude 에이전트가 그 안에서 계속 돌아가는"** 구조로 가는 방향.
- AgentCore CLI 문서를 흘어보면, Claude Code 로컬 스킬과 서버사이드 에이전트의 차이를 이해하는 데 도움이 돼. 지금 공부해두면 6개월 뒤에 자연스럽게 써먹을 수 있어.

### 2. Google: Gemma 4 오픈소스 모델 + Gemini Flash TTS + 요금제 개편

**📌 무슨 일?**

Google이 4월 27일 Gemini API 업데이트로 세 가지를 동시에 내놨어:

- **Gemma 4 모델 출시** (`gemma-4-26b-a4b-it`, `gemma-4-31b-it`): AI Studio와 Gemini API에서 바로 쓸 수 있는 오픈소스 모델. 26B와 31B 두 버전.
- **Gemini 3.1 Flash TTS Preview**: 저비용·고표현력 텍스트→음성 변환 모델. 음성 톤·감정을 API 파라미터로 조절 가능.
- **Flex·Priority 추론 티어 신설**: 비용 절감(Flex) 또는 응답 속도 우선(Priority) 중 하나를 선택할 수 있는 새 요금제 구조.

**🤔 왜 중요?**

Gemma 4는 Google이 직접 만든 최신 오픈소스 모델이야. 26B~31B 파라미터는 MacBook Pro에서도 로컬 실행이 가능한 크기야.

DeepSeek V4, Qwen 3.6과 같은 주에 나온 게 우연이 아니야 — 오픈소스 모델 경쟁이 최상위 구간까지 올라온 거야.

**👉 너에게 영향**

- **Gemma 4 로컬 실행 고려**: 무료 원칙에 가장 부합하는 모델 중 하나. `ollama pull gemma4:26b` 형태로 로컬 실행 가능(사양 확인 필요).
- **Gemini Flash TTS**: 사이드 프로젝트에 음성 기능 추가할 때 저렴한 선택지. 예: 앱 알림 음성, 튜토리얼 나레이션.
- **Flex 티어**: 실시간 응답이 필요 없는 배치 작업(리서치·요약·테스트 생성)에 비용을 크게 낮출 수 있어.

### 3. Qwen 3.6 출시 + DeepSeek V4 세계 모델 심층 분석 (MIT Technology Review)

**📌 무슨 일?**

두 가지 오픈소스 AI 관련 뉴스가 4월 27일 나왔어:

- **Qwen 3.6 출시** (Alibaba): Qwen 3 시리즈 업그레이드 버전. Apache 2.0 오픈소스 — 상업 프로젝트 무료 사용 허용.
- **MIT Technology Review 심층 기사** (Apr 27): DeepSeek V4가 단순 언어 모델이 아니라 **"세계 모델(world model)"** 접근법을 쓴다는 분석. 텍스트·코드·수학·과학을 하나의 내부 세계 표현으로 통합하는 방식.

**🤔 왜 중요?**

Qwen 3.6은 DeepSeek V4, Kimi K2.6과 함께 "중국발 오픈소스 3대장" 구도를 완성시켜.

MIT TR의 DeepSeek 분석은 단순 벤치마크를 넘어서 — "이 모델이 왜 이렇게 효율적인가?"에 대한 기술적 배경을 처음으로 공개적으로 설명한 기사야.

**👉 너에게 영향**

- **실용 관점**: Qwen 3.6은 한국어·중국어·영어 3개 언어에서 상위권 성능 유지. 한국어 작업에서 DeepSeek보다 나은 편.
- **이해 관점**: MIT TR 기사를 읽으면 "왜 DeepSeek이 1.6T 파라미터인데 49B만 켜도 GPT-5.5급이 되는지"를 이해할 수 있어. 개발자가 아니어도 읽을 만한 수준.

## 🔬 트렌드 한 컷: "인프라 전쟁"이 시작됐다

지금까지 AI 경쟁은 **"누가 더 똑똑한 모델을 만드느냐"**였어. 4월 27일 뉴스는 그게 달라지고 있다는 신호야.

**비유로 설명**: 스마트폰 초기 시대엔 모두 "어떤 폰이 더 좋은 카메라?"를 논했어. 그런데 어느 순간 핵심 경쟁이 **"앱스토어"**로 옮겨갔어 — 아무리 좋은 폰도 앱 생태계가 없으면 소용없으니까.

AI도 지금 같은 전환점에 있어:

- **AWS**: AgentCore CLI + Claude Cowork Bedrock → "내 클라우드에서 에이전트를 운영하세요"
- **Google**: Flex/Priority 티어 + Gemma 4 오픈소스 → "내 인프라에 AI를 내재화하세요"
- **Anthropic**: Managed Agents 베타 + 시드니 오피스 → "전 세계 기업에 에이전트를 서비스로 납품"

모델 벤치마크 경쟁에서 **누가 기업 에이전트 인프라의 표준이 되느냐** 경쟁으로 무게중심이 이동하는 중.
