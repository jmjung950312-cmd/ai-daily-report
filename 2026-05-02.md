Anthropic은 명시적으로 포함되지 않았어. 이유는 지금까지 계속된 그 논란이야:

**🤔 왜 중요?**

세 가지 층위에서 중요해:

1. **법적 역설** — 연방 판사가 이미 "공급망 리스크 지정에 법적 근거 불충분"으로 임시 금지 명령을 내린 상태야. 그런데 Pentagon은 법원 명령과 무관하게 경쟁사들과 계약을 진행했어. "차단은 못 하지만 우리는 안 써" 전략으로 선회한 거야.

3. **Mythos 변수** — Pentagon CTO Emil Michael이 CNBC에 "Anthropic 블랙리스트는 유지하지만 Mythos(사이버 취약점 발굴 AI)는 별도 국가 안보 문제"라고 발언했어. Mythos 활용 협상이 별도 트랙으로 진행될 가능성을 열어둔 셈이야.

**👉 너에게 영향**

- **Claude Code 사용에 영향 없음**: 이 사건은 연방 정부 계약 레이어의 이야기야. 개인·기업용 Claude API·Claude Code는 정상 사용 가능해.
- **장기 관점**: Anthropic이 원칙을 유지하며 정부 계약을 잃을 경우, 수익 기반을 민간·기업 시장에서 더 공격적으로 확장해야 해. Claude Security 론칭, 크리에이티브 툴 파트너십 확대가 그 전략의 일부로 읽혀.

### 3. Claude Code 신기능 묶음 — ExitWorktree, 크론 킬 스위치, 그 외

**📌 무슨 일?**

4월 말~5월 1일 사이 Claude Code에 다수의 신기능이 배포됐어:

| 기능 | 내용 |
|---|---|
| **ExitWorktree 툴** | EnterWorktree로 진입한 격리 환경에서 빠져나오는 툴. 워크트리 사이클 완성. |
| **CLAUDE_CODE_DISABLE_CRON** | 환경변수 한 줄로 현재 세션의 /loop·cron 작업 즉시 중단 |
| **/copy "w" 키** | 출력 내용을 파일로 바로 저장. SSH 환경에서 클립보드 없이 사용 가능 |
| **`/plan` 설명 인수** | `/plan [설명]` 형태로 플랜 생성 시 컨텍스트 추가 가능 |
| **노력도 단순화** | low / medium / high 세 단계로 정리 (max 제거) |
| **CLAUDE.md HTML 주석 숨김** | HTML 주석(`<!-- -->`)이 Claude에게 자동 주입 시 보이지 않음 |
| **bash 자동 승인 확장** | lsof, pgrep, tput, ss, fd, fdfind 명령 허용 목록 추가 |
| **`claude project purge`** | 프로젝트의 Claude Code 상태 데이터 일괄 삭제 |

**🤔 왜 중요?**

이번 묶음에서 가장 실용적인 변화 둘을 짚으면:

**CLAUDE_CODE_DISABLE_CRON**: /loop나 /schedule로 반복 작업을 돌리다가 갑자기 멈춰야 할 때 기존엔 프로세스를 강제 종료해야 했어. 이제 환경변수 한 줄로 우아하게 정지 가능.

**ExitWorktree**: EnterWorktree(격리 브랜치에서 작업)와 ExitWorktree(복귀)가 짝을 이루게 됐어. 이전엔 격리 환경을 벗어나는 게 불명확했는데, 이제 명시적 도구로 관리 가능.

**👉 너에게 영향**

- **즉시 활용**: CLAUDE_CODE_DISABLE_CRON은 /loop를 쓰는 사람이라면 세션 설정에 넣어두면 유용해. 비용 초과나 작업 오류 시 빠른 정지 가능.
- **SSH 사용자**: /copy의 "w" 키는 리모트 서버에서 Claude Code를 쓸 때 결과를 파일로 빼는 실질적 방법이야.
- **노력도 단순화**: max가 사라지고 high가 최대값이 됐어. 기존에 max를 쓰던 워크플로우를 high로 교체하면 돼.

### Mythos + Glasswing — 세상에서 가장 위험한 AI가 제한 공개된 이유

4월 7일 Anthropic이 공개한 **Claude Mythos Preview**의 실제 수치가 이번에 더 넓게 알려졌어.

**수치로 보는 Mythos의 능력**:

| 항목 | Opus 4.6 | Mythos Preview |
|---|---|---|
| Firefox JS 엔진 취약점 → 익스플로잇 성공 횟수 (수백 회 시도) | **2회** | **181회** |
| 레지스터 제어 달성 | 0회 | **29회 추가** |
| 미발견 버그 (모든 주요 OS·브라우저) | N/A | 수백 개 (99%+ 미패치 상태) |

Mythos가 찾은 대표 사례: **CVE-2026-4747** — 17년간 모르고 있던 FreeBSD NFS 원격 코드 실행 취약점. 인터넷에 연결된 모든 NFS 서버에 인증 없이 root 권한을 얻을 수 있는 버그를 **완전 자율로** 발굴하고 익스플로잇했어.

비유하면 — 잠긴 문을 여는 열쇠 복제 기계가 있는데, 그 기계가 세상 모든 자물쇠의 취약점을 스스로 찾아내고 17년 전에 만들어진 잘못된 패턴을 포착하는 수준이야.

**왜 공개 안 하나?** 발견된 취약점의 99% 이상이 아직 패치가 안 됐어. 공개하면 해커들이 즉시 악용 가능.

**Glasswing 컨소시엄**: Anthropic이 Mythos 접근을 허용한 40개 이상 기관. Google, Microsoft, AWS 등 대형 기업·보안사들이 포함됐어. 모니터링된 환경에서만 사용 가능. 악용 방지 조건으로 접근 허가.

## 🔬 트렌드 한 컷: "안전 원칙"이 정부 계약의 킬러 변수가 된 첫 번째 날

오늘 Pentagon 발표는 AI 산업 역사에서 기록될 사건이야.

이전까지 AI 기업들의 정부 계약 성사/실패 변수는 이런 것들이었어:
- 기술력 (모델 성능 벤치마크)
- 가격
- 보안 인증 (FedRAMP, IL 등)
- 기존 관계

**5월 1일 이후엔 새 변수가 공식화됐어**: "자사 AI를 어떤 용도에도 쓰게 해줄 의사가 있는가?"

비유하면 — 무기 제조사에 "이 총은 민간인 대량 학살에도 팔 수 있어?"라고 묻는 거야. 이전엔 그 질문 자체가 없었어.

Anthropic은 "아니오"라고 했고, OpenAI·Google·AWS는 "예"라고 했어 (또는 조건부로 동의했어). 결과: 정부 계약 시장 8개 참여사 vs. 0개.

**이게 왜 구조적 변화인가?**

두 가지 가능한 미래가 동시에 열렸어:

- **시나리오 A**: Anthropic이 수익 기반을 민간·기업으로 유지하면서도 장기적으로 "안전 AI의 표준"을 선점하는 포지션을 확보
- **시나리오 B**: 안전 원칙을 타협하지 않으면 자금·인재·정부 계약에서 밀리면서 결국 원칙 자체를 수정하는 압력을 받는 구조

2026년 AI 산업이 이 두 시나리오 중 어느 쪽으로 이동하는지를 추적하는 게 앞으로 보고서의 핵심 트렌드가 될 거야.

### ⏱️ 15분으로 시도

1. **CLAUDE_CODE_DISABLE_CRON 세팅 준비**: `~/.claude/settings.json`에 `CLAUDE_CODE_DISABLE_CRON` 사용법을 메모해둬. /loop를 쓰다 중단이 필요할 때 환경변수로 즉시 정지 가능해. (지금 설정할 필요 없음 — 필요할 때 `export CLAUDE_CODE_DISABLE_CRON=1` 한 줄)

2. **Grok 4.3 API 가격 비교 북마크**: 사이드 프로젝트에 비디오 입력이나 실시간 X 데이터가 필요한 기능이 있다면, Grok 4.3 API($1.25/$2.50)와 Gemini Flash API 가격을 비교해봐. Claude가 유일한 선택지가 아닌 시대야.

### 🕐 1시간으로 시도

3. **ExitWorktree 워크플로우 실험**: Claude Code에서 `EnterWorktree` → 작업 → `ExitWorktree` 사이클을 한 번 직접 실행해봐. 격리 브랜치에서 안전하게 실험하고 돌아오는 패턴이 체득되면, 위험한 리팩토링이나 큰 기능 추가 시 "이 브랜치가 망해도 상관없어" 마인드셋으로 더 과감하게 시도할 수 있어.

4. **`/copy` w 키 테스트**: SSH로 원격 서버에서 Claude Code를 쓰는 세팅이 있다면, /copy 실행 후 "w" 키를 눌러서 파일로 저장하는 방법을 확인해봐. 클립보드 없는 환경에서의 결과 추출이 훨씬 편해져.

### 📅 하루로 시도

5. **Claude Security 사용 가능 여부 확인 및 대기 등록**: Claude Enterprise 구독 중이라면 Claude Security 베타 접근을 확인해봐. 아직 Enterprise가 없다면 Team/Max 출시 대기 명단에 이름을 올려두거나, Anthropic 공식 페이지에서 관심 등록. 현재 사이드 프로젝트의 코드베이스에 보안 취약점 스캔을 한 번이라도 돌려보는 게 목표야. (무료 대안: Semgrep OSS, CodeQL이 현재 사용 가능)
