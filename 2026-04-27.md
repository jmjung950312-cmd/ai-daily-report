### 1. Claude Code 7회 연속 업데이트 (2.1.113 → 2.1.119)

**📌 무슨 일?**
4월 19일 ~ 24일 사이에 Claude Code(터미널에서 쓰는 너의 메인 도구)가 7번 업데이트됐어. 단순 버그 수정이 아니라 **사용 방식 자체가 바뀌는 신기능들**이 많아.

**🤔 왜 중요?**
이전엔 한 번에 하나의 에이전트만 돌릴 수 있었는데, 이제 **"포크된 서브에이전트(forked subagent)"**가 도입됐어. 쉽게 말하면 — 이전엔 비서 1명에게 일을 맡겼다면, 이젠 비서가 자기 분신 여러 개를 만들어 병렬로 일을 처리해. 또 `/cost`와 `/stats`가 `/usage` 하나로 통합됐고, **사용자 정의 테마**, **커스텀 PR URL**, **MCP 서버 동시 연결(기본값)** 등 — 한 줄 한 줄이 너의 시간을 줄여줘.

**👉 너에게 영향 (Claude Code 터미널 사용자 관점)**
- **즉시 시도**: 터미널에서 `claude --version`으로 2.1.119 이상인지 확인. 아니면 자동 업데이트 대기 또는 재시작.
- **꿀팁**: `/usage` 명령어로 이번 달 토큰·비용을 한 화면에서 본다 (이전엔 두 명령어 따로 실행).
- **고급 활용**: `CLAUDE_CODE_FORK_SUBAGENT=1` 환경변수를 넣으면 너의 `/sprint`나 `/deep-sprint` 같은 다중 에이전트 스킬이 더 빠르게 돌아간다 (실험 기능이니 한 프로젝트에서 먼저 테스트 권장).
- **신규 보안 옵션**: `sandbox.network.deniedDomains` 설정으로 Claude가 특정 도메인에 접속하지 못하게 막을 수 있음 — 외부 API 호출 실수 방지에 유용.

### 너의 워크플로우에 즉시 적용할 만한 5가지

1. **`/usage` 명령어 — 비용 추적 통합**
   터미널에서 `/usage` 한 번이면 토큰 사용량과 비용을 동시에 본다. 무료 원칙 사용자에게 매일 체크할 가치 있음.

2. **`CLAUDE_CODE_FORK_SUBAGENT=1` — `/sprint` 가속**
   너의 `/sprint`, `/deep-sprint` 스킬이 여러 에이전트를 띄울 때 **병렬 처리**가 가능해짐. 한 사이드 프로젝트에서 테스트 후 글로벌 적용 권장.

3. **`sandbox.network.deniedDomains` — 보안 강화**
   `~/.claude/settings.json`에 `"sandbox": { "network": { "deniedDomains": ["analytics.example.com"] } }` 같은 형식으로 추가. 의도치 않은 트래킹 호출 차단.

4. **커스텀 테마 — `/theme`**
   `~/.claude/themes/`에 JSON 만들면 너만의 색상 테마 적용. VS Code IDE 통합에서도 동기화 가능성 (다음 업데이트 주시).

5. **`duration_ms` 훅 입력값**
   네 글로벌 룰(post-write-lint 훅 등)에서 도구 실행 시간을 알 수 있게 됨. **느린 도구를 자동 감지·경고**하는 훅 작성 가능.

## 🔬 트렌드 한 컷: "AI 에이전트의 신뢰성 문제"

**비유로 설명**: 지금까지 AI는 **"질문 잘 받는 똑똑한 인턴"**이었어. 그런데 **에이전트(agent)**는 **"혼자 일을 끝까지 처리하는 신입사원"**이야. 문제는 — 신입사원이 5단계 일을 받으면 1단계에서 잘못 이해하면 5단계 모두 틀리게 마무리해버려.

이번 주 발표된 연구들은 한 방향을 가리켜:
- **자기 점검 루프(internal feedback loop)**: 모델이 자기 답을 스스로 검증하고 고치는 방식. Claude Opus 4.6에 적용해보니 복잡한 작업에서 **20% 빨라짐**(에러 수정 사이클 감소).
- **Stanford AI Index 2026**: AI가 단순 질문 답변엔 강하지만, **판단이 필요한 작업에선 여전히 인간 우위**. Nature(Apr 22)에선 "복잡한 과학 작업에서 인간 과학자가 최고 AI 에이전트를 압도"라고 보도.

**결론**: 에이전트는 **잘 정의된 작업**(체크리스트가 있는 일)에선 슈퍼파워, **모호한 판단**(설계 결정, 사람 마음 읽기)에선 여전히 너의 영역.
