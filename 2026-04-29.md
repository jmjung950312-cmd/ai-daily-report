### 1. Anthropic "Claude for Creative Work" — 8개 크리에이티브 툴 커넥터 공개

**📌 무슨 일?**

Anthropic이 4월 28일 "Claude for Creative Work"를 발표했어. 핵심은 다음 8개 크리에이티브 소프트웨어와의 공식 MCP 커넥터야:

| 툴 | 용도 | Claude가 할 수 있는 것 |
|---|---|---|
| **Ableton** | 음악 프로덕션 | 공식 문서 기반 제작 가이드·자동화 |
| **Adobe Creative Cloud** | Photoshop·Premiere 등 50+ 앱 | 배치 처리·스크립트·워크플로우 자동화 |
| **Affinity by Canva** | 디자인 | 배치 처리·레이어 관리 자동화 |
| **Autodesk Fusion** | 3D 설계 | 대화형 3D 모델 생성 |
| **Blender** | 3D·애니메이션 | Python API 자연어 인터페이스 |
| **Resolume Arena/Wire** | 라이브 퍼포먼스 비주얼 | 실시간 비주얼 컨트롤 |
| **SketchUp** | 건축·공간 3D | 대화형 3D 모델링 스타터 |
| **Splice** | 음악 샘플 | 로열티 프리 샘플 검색·추천 |

Anthropic은 또한 미술·디자인 대학 세 곳 — **Rhode Island School of Design(RISD), Ringling College, Goldsmiths University(런던)** — 과 교육 파트너십을 체결해 창작 커리큘럼에 Claude를 통합하기로 했어.

**🤔 왜 중요?**

지금까지 Claude의 주된 사용자는 개발자·데이터 분석가·기획자였어.

이번 발표는 **디자이너·음악가·영상 편집자·3D 아티스트**를 정식 타깃으로 선언한 거야. 비유하면 — 엑셀 전용으로 쓰던 AI 조수가 Photoshop·Ableton 자리에도 나타난 것.

기술적으로도 중요한 점이 있어: 모든 커넥터가 **MCP(Model Context Protocol)** 기반이야. Anthropic이 만든 오픈소스 연결 표준을 자신의 제품 확장에도 일관되게 쓴다는 선언이기도 해.

**👉 너에게 영향**

- **지금 당장**: 사이드 프로젝트에 Blender나 SketchUp 작업이 있다면 Claude에서 바로 Python API를 자연어로 제어하는 실험이 가능해.
- **무료 원칙 부합**: MCP 기반이라 Claude.ai 또는 Claude Code에서 커넥터를 추가하면 별도 유료 툴 없이 사용 가능.
- **중기 관점**: Adobe CC 커넥터가 안정화되면 "Photoshop 배치 처리 + Claude" 조합이 콘텐츠 제작 워크플로우를 바꿀 수 있어.

### 2. Claude Code v2.1.121 — 메모리 누수 해결 + MCP 성능 개선

**📌 무슨 일?**

4월 28일 Claude Code v2.1.121이 출시됐어. 주요 변화:

1. **멀티 GB RSS 메모리 누수 수정** — 오래 사용할수록 메모리가 수 GB씩 쌓이던 문제 해결
2. **MCP 서버 `alwaysLoad` 옵션** — 자주 쓰는 MCP 서버를 항상 켜둬서 매번 초기화 없이 즉시 사용
3. **`/skills` 검색 박스 신규 추가** — 스킬 목록에서 타이핑으로 바로 필터링
4. **PostToolUse 훅에서 툴 출력 교체** — 훅이 도구 실행 결과를 가로채 다른 값으로 바꾸기 가능 (자동 후처리용)
5. **작업 디렉토리 삭제 시 Bash 툴 무력화 버그 수정** — 세션 중 폴더를 지우면 Bash가 먹통 되던 문제 수정
6. **OAuth 신뢰성 개선** — 멀티 플랫폼 인증 불안정 해소

**🤔 왜 중요?**

v2.1.119(Apr 24) 이후 4일 만의 대형 업데이트야.

특히 메모리 누수 수정은 — Claude Code를 하루 종일 켜두는 사용자라면 느꼈을 체감 문제야. 오래 켜두면 맥이 느려지거나 팬이 돌던 원인 중 하나가 이거였어.

MCP `alwaysLoad`는 Claude Code 스킬 시스템을 쓰는 사람에게 실질적인 속도 향상을 의미해 — 매 스킬 실행마다 MCP 서버가 다시 시작되는 지연이 사라지거든.

**👉 너에게 영향**

- **즉시 업데이트 권장**: `claude --version`으로 2.1.121 확인. 메모리 누수 수정만으로도 업데이트 가치 충분.
- **`alwaysLoad` 활용**: 너의 `~/.claude/settings.json`에서 자주 쓰는 MCP 서버(Supabase, GitHub, Playwright 등)에 `alwaysLoad: true` 추가하면 스킬 실행이 더 빨라져.
- **PostToolUse 훅 가능성**: 도구 실행 후 자동으로 lint 결과를 정리하거나, 파일 저장 후 자동 포맷을 적용하는 훅을 만들 수 있어. 기존보다 더 정교한 자동화 가능.

### 3. Grok 4.3 Beta — 비디오 입력·슬라이드·PDF 생성 공개

**📌 무슨 일?**

xAI가 4월 17일 조용히 출시한 Grok 4.3 Beta의 세부 스펙이 4월 28일 공식 채널을 통해 알려졌어.

새로 추가된 기능:

1. **네이티브 비디오 입력** — 텍스트·이미지에서 비디오 이해로 멀티모달 확장
2. **프레젠테이션 슬라이드 직접 생성** — 채팅 안에서 PowerPoint 덱 바로 출력
3. **PDF·스프레드시트 다운로드** — 대화 결과를 즉시 파일로 저장
4. **멀티에이전트 아키텍처** — X(트위터) 실시간 데이터와 결합한 실시간 인텔리전스

**🤔 왜 중요?**

GPT-5.5가 코딩·추론에서 강점을 내세운다면, Grok 4.3은 "실시간 소셜 데이터 + 다양한 출력 포맷"이라는 차별화 포지션을 굳히고 있어.

다만 현재 접근 조건이 까다로워: **$300/월 SuperGrok Heavy 티어 전용**이야. 일반 사용자는 5월 말 이후 전체 롤아웃 예정.

**👉 너에게 영향**

- **비용 측면**: $300/월은 무료 원칙과 맞지 않아. 지금은 관망하고 무료 티어 롤아웃 시점에 재평가.
- **출력 형식**: 슬라이드·PDF 직접 생성 기능은 여러 AI에서 곧 표준이 될 방향. Grok이 먼저 선점했지만 Claude·GPT에서도 유사 기능이 나올 가능성 높아.

## 🔬 트렌드 한 컷: AI가 전문 소프트웨어 안으로 들어오는 날

지금까지 AI를 쓰는 방식은 주로 이랬어:

> "ChatGPT·Claude 열고 → 내가 원하는 걸 설명하고 → 결과를 복사해서 → 내 소프트웨어에 붙여넣기"

**비유하면 — 통역사를 고용해서 일을 시키는데, 통역사가 내 사무실 밖에 있어서 매번 달려가야 하는 구조야.**

Anthropic의 "Claude for Creative Work"는 이 구조를 바꿔:

> "Ableton·Photoshop·Blender 안에서 → Claude를 바로 호출하고 → 결과가 그 자리에서 반영"

이건 단순한 기능 추가가 아니야. **AI가 "별도 앱"에서 "내 도구 안의 내장 기능"으로 전환하는 첫 번째 대규모 파동**이야.

역사에서 비슷한 전환점이 있었어:
- 맞춤법 교정 프로그램이 Word 안으로 들어왔을 때
- 지도 앱이 다른 앱에 API로 내장됐을 때

AI도 지금 같은 전환점에 있어. 2년 뒤에는 "AI 창을 따로 열어서 프롬프트를 넣는다"는 개념이 낯설어질 가능성이 높아.

## 💡 활용 아이디어

### ⏱️ 15분으로 시도

1. **Claude Code 업데이트 확인**: `claude --version`으로 v2.1.121 이상인지 확인. 업데이트 후 메모리 사용량 체감 비교.

2. **Claude for Creative Work 커넥터 탐색**: Claude.ai에서 Blender 또는 Adobe CC 커넥터가 추가됐는지 확인. MCP 커넥터 추가 방법 스캔.

### 🕐 1시간으로 시도

3. **`alwaysLoad` MCP 설정 적용**: `~/.claude/settings.json`을 열고 자주 쓰는 MCP 서버에 `alwaysLoad: true` 추가. 스킬 실행 속도 전후 비교.

   ```json
   {
     "mcpServers": {
       "supabase": {
         "alwaysLoad": true
       }
     }
   }
   ```

4. **PostToolUse 훅 실험**: 파일 저장 후 lint를 자동 실행하는 PostToolUse 훅 작성. 기존 post-write-lint 훅보다 도구 출력 기반으로 더 정밀한 조건 분기 가능.

### 📅 하루로 시도

5. **Claude + Blender 연동 테스트**: Blender MCP 커넥터를 연결하고 자연어로 간단한 3D 씬을 만들어봐. 사이드 프로젝트 프로토타입에 3D 시각화가 필요한 부분이 있다면 이 조합이 진입 장벽을 크게 낮춰줄 수 있어.
