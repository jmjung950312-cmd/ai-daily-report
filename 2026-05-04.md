## ⭐ 핵심 변화 TOP 3

### 1. Mistral Medium 3.5 — 128B로 채팅·추론·코딩을 한 모델에 담다

**📌 무슨 일?**

5월 2일, 프랑스 AI 스타트업 Mistral이 **Mistral Medium 3.5**를 발표했어.

핵심 스펙:

| 항목 | 내용 |
|---|---|
| **파라미터** | 128B (Dense, MoE 아님) |
| **컨텍스트 창** | 256k 토큰 |
| **SWE-Bench Verified** | 77.6% |
| **출시 위치** | Vibe, Le Chat 기본 모델로 교체 |
| **역할** | 채팅·추론·코딩 세 가지를 단일 가중치로 처리 |

기존 Mistral 모델들은 용도별로 나뉘어 있었어 — Devstral은 코딩 전용, Mistral Large는 채팅 중심. Medium 3.5는 이 셋을 하나의 가중치 세트로 묶었어.

**🤔 왜 중요?**

세 가지로 짚을게:

1. **SWE-Bench 77.6%의 의미** — SWE-Bench Verified는 실제 GitHub 이슈를 해결할 수 있는지를 테스트하는 코딩 벤치마크야. 77.6%는 GPT-5.5 수준과 비슷하게 경쟁력 있는 숫자야. "유럽 AI가 코딩에서 처음으로 프론티어 레벨에 진입했다"는 평가가 나왔어.

2. **Dense 모델 선택** — 요즘 대형 모델들은 대부분 MoE(Mixture of Experts — 필요한 부분만 켜는 구조)를 써. 메모리와 비용을 아끼기 위해서야. Mistral은 Dense(모든 파라미터를 항상 사용)를 선택했는데, 이건 추론 품질이 더 일관되고 예측 가능하다는 장점이 있어.

3. **가격 논란** — API 가격이 공개되자 "중간 모델치고 비싸다"는 반응이 일부 나왔어. 정확한 가격은 출시 페이지에서 확인 필요.

**👉 너에게 영향**

- **Claude API 대안 탐색**: 코딩 작업에서 SWE-Bench 기준으로 비슷한 성능이라면 가격 비교 테스트 해볼 가치가 생겼어. Mistral API를 통해 직접 호출 가능.
- **Le Chat 무료 사용**: Mistral의 Le Chat 앱은 Medium 3.5를 기본 모델로 탑재했어. 무료 티어에서도 사용 가능 — 사이드 프로젝트 아이디어 탐색에 바로 써볼 수 있어.
- **무료 원칙 OK**: Le Chat는 무료 플랜 존재. Mistral API는 유료이지만 Claude API와 동일하게 선택적 사용 가능.

### 2. Mistral Le Chat Work Mode + Vibe Remote Agents — 에이전트가 클라우드에서 일한다

**📌 무슨 일?**

Medium 3.5 발표와 동시에 Mistral이 두 가지 에이전트 기능을 공개했어:

**Le Chat Work Mode:**

- Mistral의 소비자용 채팅 앱 Le Chat에 **멀티스텝 병렬 에이전트** 탑재
- 이메일, 캘린더, 문서, Jira, Slack을 동시에 조회하고 처리
- 민감한 작업 전엔 사용자 명시적 승인 요청
- 모든 툴 호출과 추론 단계를 화면에 표시 (투명성)

**Vibe Remote Agents:**

- 코딩 에이전트 Vibe의 작업을 **클라우드에서 비동기 실행**
- CLI나 Le Chat에서 세션을 시작하면 격리된 샌드박스에서 백그라운드 실행
- 로컬 세션을 클라우드로 "텔레포트" — 세션 히스토리와 작업 상태를 잃지 않고 이동 가능

**🤔 왜 중요?**

비유하면 — Claude Code가 "내 PC에서 대화하면서 코딩"이라면, Vibe Remote Agents는 "내가 PC를 닫아도 서버에서 혼자 일하다가 끝나면 알려줘"야.

두 기능이 합쳐지면:

1. Le Chat에서 "이 Jira 이슈 다 처리하고 Slack에 보고해줘"라고 하면 Work Mode가 처리
2. 코딩 작업은 Vibe에서 시작하고, 돌아가는 동안 다른 일을 함

이건 Claude Code의 `/loop`·`/schedule`과 직접 경쟁하는 기능이야. Mistral이 에이전트 레이어에서도 프론티어 경쟁에 진입했다는 신호야.

**👉 너에게 영향**

- **Le Chat Work Mode 실험 가능**: 현재 Pro(€14.99/월) 이상 또는 무료 티어 일부에서 베타 접근 가능 [unverified: 정확 플랜 조건 확인 필요]
- **Claude Code와 비교**: /loop와 Vibe Remote Agents를 같은 반복 작업에 써보면 두 도구의 "비동기 지속성" 차이를 직접 체감할 수 있어.

## 🔬 트렌드 한 컷: Dense vs MoE — 왜 Mistral은 "모든 뇌세포를 항상 켜는" 쪽을 택했나

요즘 대형 AI 모델들은 대부분 **MoE(Mixture of Experts)** 구조를 써. 비유하면 "1000명의 전문가가 있는데, 질문이 들어오면 그 중 필요한 50명만 깨워서 답한다"는 방식이야. 비용을 아끼면서 파라미터 수를 늘릴 수 있어.

Mistral Medium 3.5는 **Dense** 구조야. "1000명이 항상 다 깨어 있다."

왜 이 선택을 했나:

- **일관성**: 어떤 질문이 오든 항상 같은 품질. MoE는 어떤 전문가가 "깨어났느냐"에 따라 품질이 미묘하게 달라질 수 있어.
- **예측 가능성**: 기업 고객이나 코딩 에이전트처럼 "항상 같은 수준을 요구"하는 용도에 Dense가 유리해.
- **추론의 연속성**: 긴 다단계 추론에서 MoE보다 Dense가 더 일관된 사고 흐름을 유지한다는 연구 결과가 있어 [unverified].

트레이드오프:

| | Dense | MoE |
|---|---|---|
| 추론 품질 일관성 | 높음 | 중간 |
| 비용 (API 호출) | 높음 | 낮음 |
| 대규모 파라미터 확장 | 제한적 | 용이 |
| 훈련 복잡도 | 낮음 | 높음 |

Mistral의 선택은 "코딩 에이전트처럼 신뢰성이 중요한 용도에서 Dense가 더 맞다"는 판단이야. 이 결정이 옳은지는 6개월 뒤 실제 사용 데이터가 말해줄 거야.
